Git Commands and their uses

Git is a version control system that records changes to a file or set of files over time so that we can recall specific version later and save them on a remote repository like Github.

To use this version control system first we need to installed it in our local system.

Repository is a directory or storage space where our projects can live

Below are the git commands and their corresponding functionalities:

  • git --version

    It is used to check the git version installed in our system.

  • git init

    Initialize a new, empty repository for our project in our working directory.

  • git config --global user.email "you@example.com"

    Used to set the user email id which is used as identification while commiting the code

  • git config --global user.name "Your Name"

    Used to set the user name which is used as identification while commiting the code

  • git status

    Used to know the status of our code files,whether they are commited,staged or untracked.

  • git add [filename]

    Used to add untracked code file, means file from local sytem moved to repository .

  • git add .

    Used to add all the code files from local system to repository .

  • git commit -m "[message]"

    Commit messages are a way of communication between team members, where message is given to identify the recent changesor making a log of the changes.

  • git commit -m "[message]"

    Commit messages are a way of communication between team members, where message is given to identify the recent changesor making a log of the changes.

  • git log

    Used to check the list of commits we have done so far.

  • git reset --hard "[SHA number]"

    [SHA number is Secured Hash Algorithm number.It is unique for each commit.] It is used to delete recent commit changes and go back to the previous commit working code by that particular SHA number.

  • git reset --soft[SHA number]

    Used to go back to the previous com ited code along with the latest code change still available. Git log will only show the previous commited logs but the recent commited log will not be there even chnages will be available in this commit.

  • git remote add origin [url]

    Used to set the origin location on remote repository where the code needs to pushed from local repository.

  • git branch -M main

    Sets the branch name as main.

  • git push -u origin main

    Used to push the code from local system to remote repository at the specified URLs origin's branch.

  • git clone url

    Used to copy the code from remote to our local system.

  • git pull

    used to fetch and download content from a remote repository and immediately update the local repository to match that content.

  • git branch

    Used to check which are the branches and which is the current branch.

  • git branch [branchname]

    Used to create a branch with the mentioned branch name.

  • git merge branchname

    It joins branches. It combines multiple sequences of commits into one.

  • git checkout branchname

    Used to checkout from current branch and move on to the mentioned branch.
